## 🎯 What Is GridSense AI?

GridSense AI is built around a single principle: **technology should empower humans, not confuse them.**

Traditional power quality monitors show operators cryptic numbers like *"THD: 12.6%"* or *"PF: 0.78"*. A factory operator without an electrical engineering degree has no idea what these mean or what to do about them. So problems go unnoticed. Equipment fails. Sometimes there are fires.

GridSense AI changes that. It:

1. **Detects** power quality faults (Sag, Swell, Harmonics, Transients) with **99.16% accuracy**
2. **Explains** them in plain human language
3. **Identifies** the most likely root cause
4. **Recommends** specific corrective actions
5. **Calculates** financial loss in ₹ per hour

Instead of *"THD: 12.6%"*, the operator sees:  
*"Harmonics detected. Caused by VFDs. Install harmonic filters. You're losing ₹1,440/hour."*

This is **Industry 5.0** — AI that empowers humans instead of replacing them.

## 🧠 Technical Approach

### Dataset
- **Source:** Synthetic dataset generated via `generate_dataset.py` (physics-based simulation)
- **Samples:** 15,000 samples (3,000 per class)
- **Classes:** Normal, Voltage Sag, Voltage Swell, Harmonics, Transients
- **Quality:** Zero missing values, perfectly balanced class distribution (3,000 each)

### Features (9 Total)

**8 Raw:**  
RMS_Voltage • Peak_Voltage • THD • Duration • DWT_Energy_Levels • DWT_Entropy • Signal_Noise_Ratio_dB • Phase

**1 Engineered (Physics-Based):**
- `Crest_Factor = Peak / RMS` — detects transient spikes (ideal sine value: √2 ≈ 1.414)

### Normalization
**StandardScaler (Z-Score):** `z = (x - μ) / σ`  
Every feature normalized to mean=0, std=1 to prevent scale bias.

### Model
**Random Forest Classifier** — 200 decision trees  
Hyperparameters: `max_depth=20`, `min_samples_split=5`, `min_samples_leaf=2`, `max_features='sqrt'`

**Why Random Forest?**
- Trains in 2 seconds (no GPU required)
- Interpretable (feature importance)
- Robust ensemble approach (200 trees vote)
- Optimal for tabular data with 15,000 samples
- No overfitting with proper validation
